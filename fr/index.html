Le concept

Qu'est-ce qu'un agent IA ?

Un agent IA n'est pas un chat-bot. C'est un collaborateur numérique avec son propre périmètre de responsabilité, sa propre mémoire et un accès direct à vos systèmes informatiques.

Ce que fait un chat-bot
  • × Répond aux questions à partir de son matériel d'entraînement
  • × Oublie tout après la conversation
  • × Ne peut rien faire — ne génère que du texte
  • × Attend passivement votre saisie
Ce que fait un agent AIMOS
  • Supervise son périmètre de responsabilité — stocks, factures ouvertes, demandes clients, échéances
  • Agit de manière autonome — interroge les bases de données, crée des documents, envoie des messages, programme des rappels
  • Se souvient durablement — connaît chaque client, chaque préférence, chaque dossier en cours
  • Agit de manière proactive — signale les seuils critiques, rappelle les échéances, relance les demandes en attente
  • Prépare mais ne décide jamais seul — brouillons d'e-mails, propositions de réservation, demandes d'achat sont soumis au collaborateur pour approbation

Architecture des agents

La flotte de spécialistes

Chaque agent est un expert métier doté de sa propre mémoire, de ses propres outils et d'un accès direct à vos systèmes. Il supervise son périmètre de responsabilité et accomplit sa mission de manière autonome — 24 heures sur 24.

Connectivité universelle

Votre système parle SQL, REST ou SOAP ?
AIMOS comprend les trois.

Qu'il s'agisse d'une base de données relationnelle, d'une API REST moderne ou d'un logiciel legacy des années 90 : AIMOS connecte votre infrastructure informatique existante sans cloud et sans surcharge middleware.

Accès SQL direct

PostgreSQL, MSSQL, Firebird, MySQL — SELECT uniquement, pas d'écriture sur les données de production.

REST / SOAP / GraphQL

API modernes et legacy à parts égales. OAuth2, clés API, NTLM — toutes les méthodes d'authentification.

Systèmes de fichiers & SFTP

Lecteurs réseau locaux, partages SMB, SFTP via VPN — même derrière DS-Lite/NAT via Tailscale.

Canaux de communication

Telegram, e-mail (IMAP/SMTP), messages vocaux (STT/TTS) — l'agent répond là où le collaborateur pose sa question.

Compliance & Sécurité

Conçu pour les auditeurs,
pas contre eux.

Chaque décision architecturale d'AIMOS a été prise sous l'angle de la conformité réglementaire — pas ajustée après coup.

RGPD / GDPR

Localisation complète des données sur le SovereignNode. Aucun octet ne quitte votre serveur — sauf autorisation explicite. Le PII-Vault anonymise automatiquement les appels API externes.

ISO 9001 & 27001

Journalisation complète de toutes les actions des agents. Chaque appel d'outil, chaque requête LLM, chaque décision est documenté avec horodatage et identifiant de session.

Sécurité de révision

Human-in-the-Loop comme principe architectural : aucun LLM n'a d'accès direct au niveau transactionnel. Toute opération comptable requiert une approbation humaine.

Conforme GoBD

Séparation nette entre brouillon et validation. Les agents préparent les écritures comptables — la validation n'est effectuée que par des comptables autorisés dans le système cible.

Vérifiable par Journaux d'audit · Traces PII-Vault · Identifiants de session · Inspection du code source
