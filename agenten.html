Agenten-Architektur

Die Agenten-Flotte

Autonome Spezialisten mit eigenem Gedächtnis, eigenen Werkzeugen und Persönlichkeit — koordiniert durch den Orchestrator auf Ihrem SovereignNode.

Grundlagen

Was ist ein AIMOS-Agent?

Ein AIMOS-Agent ist ein autonomer Fachspezialist, der vier Kernkomponenten verbindet:

M

Memory (Gedächtnis)

Eigene SQLite-Datenbank mit semantischem, episodischem und prozeduralem Langzeitgedächtnis.

S

Skills (Werkzeuge)

Fachspezifische Tool-Sammlung: SQL-Abfragen, REST-Aufrufe, Datei-Operationen, domänenspezifische Funktionen.

L

LLM (Sprachmodell)

Lokales Modell via lokale LLM-Runtime. Der Agent baut den vollständigen Prompt aus System-Prompt, Memory-Kontext und Benutzeranfrage.

C

Connector (Anbindung)

Kommunikationskanal zum Benutzer: Telegram, E-Mail, Sprache oder Dashboard.

CR-142 — Sicherheit

Execution Rings

Drei Vertrauensstufen bestimmen, welche Werkzeuge ein Agent nutzen darf. Wie bei einem Betriebssystem-Kernel.

0

Ring 0 — Read Only

Niedrigstes Risiko. Der Agent darf nur lesen: Gedächtnis abrufen, Konfiguration einsehen. Kein Zugriff auf externe Systeme, kein Schreibzugriff.

memory_recall config_read time_now
1

Ring 1 — Standard

Lesezugriff auf externe Systeme. SQL-SELECT, REST-GET, Datei lesen. Kein Schreibzugriff auf Produktionsdaten. Standard-Ring für die meisten Agenten.

sql_query rest_get file_read memory_search brave_search
2

Ring 2 — Full Access

Vollzugriff inklusive Schreiboperationen. Erfordert explizite Freischaltung durch den Administrator. Human-in-the-Loop für kritische Aktionen.

file_write sql_execute rest_post email_send shell_exec

3-Tier Memory

Langzeitgedächtnis

Drei Gedächtnistypen, Hybrid-Suche und ein Dreaming-Zyklus zur Konsolidierung.

Token-Optimierung

Cognitive Balance

Dynamischer Ausgleich zwischen Gedächtnistiefe und Antwortlänge — innerhalb des begrenzten Kontextfensters.

Level Memory Antwort Einsatz
1 Minimal Sehr lang Analyse, Berichte
2 Reduziert Lang Zusammenfassungen
3 Ausgewogen Mittel Standard (Default)
4 Tief Kurz Kontextreiche Fragen
5 Maximum Minimal Recherche, Fakten-Check

Koordination

Inter-Agent Kommunikation

Agenten können einander über die PostgreSQL-Relay-Datenbank Nachrichten senden — mit eingebautem Ping-Pong-Limit.

Connector Families

Agenten der gleichen Connector Family teilen sich einen Kommunikationskanal. Nachrichten an die Familie werden automatisch an den zuständigen Agenten geroutet.

Ping-Pong-Schutz

Maximum 3 Hin-und-Her-Runden zwischen zwei Agenten pro Anfrage. Verhindert Endlos-Schleifen und unkontrollierten Token-Verbrauch.

Praxisbeispiele

Beispiel-Agenten

Drei vorkonfigurierte Agenten-Typen für typische Industrieumgebungen.

Konstruktions-Agent

Ring 1 — Standard

Liest AutoCAD- und Tekla-Dateien, erstellt automatisierte Stücklisten, prüft Statik-Vorgaben und erkennt Materialabweichungen. Greift auf CAD-Datenbanken per SQL und Datei-Konnektoren zu.

Skills
cad_read bom_generate sql_query file_read
Konnektoren
.dwg .ifc Tekla API BIM

Logistik-Agent

Ring 1 — Standard

Koppelt Uyumsoft, SAP, Navision und andere ERP-Systeme. Überwacht Lagerbestände in Echtzeit, meldet kritische Unterschreitungen und automatisiert Bedarfsanforderungen über SQL- und REST-Konnektoren.

Skills
sql_query rest_get stock_check alert_send
Konnektoren
Uyumsoft SAP RFC Navision REST

Finanz-Agent

Ring 1 — Standard (Schreiben: Ring 2)

Integriert ETA V8, DATEV und weitere Buchhaltungssysteme. Analysiert Cashflow, bereitet Buchungen vor und liefert tagesaktuelle Salden. Buchungs-Festschreibung nur durch autorisierte Sachbearbeiter.

Skills
sql_query cashflow_calc booking_prep datev_export
Konnektoren
ETA V8 DATEV Firebird MSSQL

Dies sind Beispiele. AIMOS-Agenten werden pro Unternehmen individuell konfiguriert — jeder Agent wird auf die spezifischen Systeme und Anforderungen zugeschnitten.
