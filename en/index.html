The Concept

What Is an AI Agent?

An AI agent is not a chatbot. It is a digital specialist with its own area of responsibility, its own memory, and direct access to your IT systems.

What a Chatbot Does
  • × Answers questions from its training data
  • × Forgets everything after the conversation
  • × Cannot take action — only generates text
  • × Waits passively for your input
What an AIMOS Agent Does
  • Oversees its area of responsibility — inventory levels, open invoices, customer inquiries, deadlines
  • Acts autonomously — queries databases, creates documents, sends messages, sets reminders
  • Remembers permanently — knows every customer, every preference, every open case
  • Acts proactively — reports critical shortages, reminds of deadlines, follows up on open requests
  • Prepares, never decides on its own — email drafts, booking proposals, requisition notices are sent to the employee for approval

Universal Connectivity

Your system speaks SQL, REST or SOAP?
AIMOS understands all three.

Whether relational database, modern REST API, or legacy software from the 90s: AIMOS connects your existing IT infrastructure without cloud dependency and without middleware overhead.

Direct SQL Access

PostgreSQL, MSSQL, Firebird, MySQL — SELECT-only, no write access to production data.

REST / SOAP / GraphQL

Modern and legacy APIs alike. OAuth2, API keys, NTLM — all auth methods supported.

Compliance & Security

Built for auditors,
not against them.

Every architectural decision in AIMOS was made with regulatory compliance in mind — not retrofitted after the fact.

GDPR

Complete data locality on the SovereignNode. No byte leaves your server — unless you explicitly allow it. PII-Vault anonymizes external API calls automatically.

ISO 9001 & 27001

Complete logging of all agent actions. Every tool call, every LLM request, every decision is documented with timestamp and session ID.

Audit-Proof

Human-in-the-Loop as an architectural principle: No LLM has direct access at the transaction level. Every booking-relevant operation requires human approval.

German Fiscal Code (GoBD)

Clean separation of draft and finalization. Agents prepare journal entries — finalization occurs only by authorized staff in the target system.

Verifiable through Audit Logs · PII-Vault Traces · Session IDs · Source Code Access
