Architettura degli agenti

La flotta di agenti

Specialisti autonomi con propria memoria, propri strumenti e personalità — coordinati dall'Orchestrator sul Suo SovereignNode.

Fondamenti

Cos'è un agente AIMOS?

Un agente AIMOS è uno specialista autonomo che combina quattro componenti fondamentali:

M

Memory (memoria)

Proprio database SQLite con memoria a lungo termine semantica, episodica e procedurale.

S

Skills (strumenti)

Raccolta di strumenti specialistici: query SQL, chiamate REST, operazioni su file, funzioni specifiche del dominio.

L

LLM (modello linguistico)

Modello locale via runtime LLM locale. L'agente costruisce il prompt completo da system prompt, contesto di memoria e richiesta dell'utente.

C

Connector (connessione)

Canale di comunicazione con l'utente: Telegram, e-mail, voce o dashboard.

CR-142 — Sicurezza

Execution Rings

Tre livelli di fiducia determinano quali strumenti un agente può utilizzare. Come in un kernel di sistema operativo.

0

Ring 0 — Read Only

Rischio minimo. L'agente può solo leggere: richiamare la memoria, consultare la configurazione. Nessun accesso a sistemi esterni, nessun accesso in scrittura.

memory_recall config_read time_now
1

Ring 1 — Standard

Accesso in lettura ai sistemi esterni. SQL-SELECT, REST-GET, lettura file. Nessun accesso in scrittura ai dati di produzione. Ring standard per la maggior parte degli agenti.

sql_query rest_get file_read memory_search brave_search
2

Ring 2 — Full Access

Accesso completo incluse operazioni di scrittura. Richiede abilitazione esplicita da parte dell'amministratore. Human-in-the-Loop per azioni critiche.

file_write sql_execute rest_post email_send shell_exec

3-Tier Memory

Memoria a lungo termine

Tre tipi di memoria, ricerca ibrida e un ciclo di Dreaming per il consolidamento.

Coordinamento

Comunicazione inter-agente

Gli agenti possono inviarsi messaggi tramite il database relay PostgreSQL — con limite ping-pong integrato.

Connector Families

Agenti della stessa Connector Family condividono un canale di comunicazione. I messaggi alla famiglia vengono automaticamente instradati all'agente competente.

Protezione ping-pong

Massimo 3 scambi andata-ritorno tra due agenti per richiesta. Impedisce cicli infiniti e consumo incontrollato di token.

Esempi pratici

Agenti di esempio

Tre tipi di agenti preconfigurati per ambienti industriali tipici.

Agente Costruzioni

Ring 1 — Standard

Legge file AutoCAD e Tekla, crea distinte base automatizzate, verifica i requisiti statici e rileva deviazioni dei materiali. Accede ai database CAD tramite connettori SQL e file.

Skills
cad_read bom_generate sql_query file_read
Connettori
.dwg .ifc Tekla API BIM

Agente Logistica

Ring 1 — Standard

Collega Uyumsoft, SAP, Navision e altri sistemi ERP. Monitora le scorte in tempo reale, segnala livelli critici e automatizza le richieste di approvvigionamento tramite connettori SQL e REST.

Skills
sql_query rest_get stock_check alert_send
Connettori
Uyumsoft SAP RFC Navision REST

Agente Finanza

Ring 1 — Standard (scrittura: Ring 2)

Integra ETA V8, DATEV e altri sistemi contabili. Analizza il flusso di cassa, prepara le registrazioni contabili e fornisce saldi aggiornati. Registrazione definitiva solo tramite operatori autorizzati.

Skills
sql_query cashflow_calc booking_prep datev_export
Connettori
ETA V8 DATEV Firebird MSSQL

Questi sono esempi. Gli agenti AIMOS vengono configurati individualmente per ogni azienda — ogni agente viene adattato ai sistemi e alle esigenze specifiche.
