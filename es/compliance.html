Registro

Audit-Trail

Cada acción de los agentes se registra completamente. El Audit-Trail captura seis tipos de eventos:

SYSTEM_PROMPT System prompt completo para trazabilidad
REQUEST Solicitud entrante del usuario
RESPONSE Respuesta completa del agente
TOOL_START Llamada a herramienta con parámetros
TOOL_OK Resultado de herramienta (éxito o error)
LLM_USAGE Consumo de tokens, modelo, latencia

Protección de datos UE

RGPD / GDPR

AIMOS cumple el RGPD no mediante adaptación posterior, sino mediante decisiones arquitectónicas — Privacy by Design.

Art. 25 — Protección de datos desde el diseño

  • Tratamiento de datos local por defecto (SovereignNode)
  • PII-Vault anonimiza antes de cada transmisión externa
  • Sin servicio en la nube en la configuración estándar
  • Minimización de datos: solo datos de contexto relevantes en el prompt LLM

Art. 32 — Seguridad del tratamiento

  • Cifrado: PostgreSQL SSL, SQLite en disco
  • Control de acceso mediante Execution Rings (3 niveles)
  • Audit-Trail completo de todos los procesos de tratamiento
  • Mapeos PII vinculados a sesión (sin almacenamiento PII persistente)

Conformidad normativa

ISO 9001 & 27001

AIMOS respalda los requisitos de ambos estándares mediante mecanismos integrados.

ISO 9001 — Gestión de calidad

  • Trazabilidad: cada decisión en el Audit-Log
  • Rastreabilidad: Session-IDs vinculan todos los eventos
  • Procesos documentados: system prompts como SOPs
  • Acciones correctivas: ciclo Dreaming aprende de errores

ISO 27001 — Seguridad de la información

  • Clasificación: Execution Rings definen niveles de acceso
  • Control de acceso: habilitación de herramientas configurable por agente
  • Registro de eventos: todos los 6 tipos de eventos capturados
  • Seguridad física: los datos no abandonan el SovereignNode

Contabilidad

Conformidad GoBD

Los GoBD (principios para la gestión y conservación adecuada de libros, registros y documentos en formato electrónico) establecen requisitos especiales para sistemas basados en IA.

AIMOS lo resuelve mediante una estricta separación arquitectónica:

Separación: borrador vs. registro definitivo

Los agentes preparan asientos contables (borrador). El registro definitivo lo realizan exclusivamente operadores autorizados en el sistema destino (ETA V8, DATEV, etc.). Ningún LLM tiene derechos de transacción.

Human-in-the-Loop

Las acciones Ring 2 (acceso de escritura) requieren habilitación explícita. Los procesos relevantes para la contabilidad siempre requieren aprobación humana — forzado arquitectónicamente, no mediante política.

Demostrabilidad

Seguridad de auditoría

Trazabilidad completa de cada decisión asistida por IA.

Token-Tracking

Cada llamada LLM se documenta con consumo exacto de tokens (Prompt + Completion), nombre del modelo y latencia.

Trail completo

Desde la solicitud del usuario hasta las llamadas a herramientas y la respuesta — cada paso es trazable con marca temporal e ID de sesión.

PII-Vault Traces

Mapeos de anonimización vinculados a sesión. Trazable qué datos se anonimizaron y des-anonimizaron y cuándo.
