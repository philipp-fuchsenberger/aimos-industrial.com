El concepto

¿Qué es un agente de IA?

Un agente de IA no es un chat-bot. Es un colaborador digital con su propio ámbito de responsabilidad, su propia memoria y acceso directo a sus sistemas informáticos.

Lo que hace un chat-bot
  • × Responde preguntas a partir de su material de entrenamiento
  • × Olvida todo después de la conversación
  • × No puede hacer nada — solo genera texto
  • × Espera pasivamente su entrada
Lo que hace un agente AIMOS
  • Supervisa su ámbito de responsabilidad — inventarios, facturas pendientes, solicitudes de clientes, plazos
  • Actúa de forma autónoma — consulta bases de datos, crea documentos, envía mensajes, programa recordatorios
  • Recuerda de forma permanente — conoce cada cliente, cada preferencia, cada expediente abierto
  • Actúa de forma proactiva — avisa sobre niveles críticos, recuerda plazos, da seguimiento a solicitudes abiertas
  • Prepara pero nunca decide solo — borradores de correo, propuestas de reserva, solicitudes de compra se envían al empleado para su aprobación

Conectividad universal

¿Su sistema habla SQL, REST o SOAP?
AIMOS entiende los tres.

Ya sea una base de datos relacional, una API REST moderna o software legacy de los años 90: AIMOS conecta su infraestructura IT existente sin dependencia de la nube y sin sobrecarga de middleware.

Acceso SQL directo

PostgreSQL, MSSQL, Firebird, MySQL — solo SELECT, sin acceso de escritura a datos de producción.

REST / SOAP / GraphQL

APIs modernas y legacy por igual. OAuth2, API-Keys, NTLM — todos los métodos de autenticación.

Sistemas de archivos & SFTP

Unidades de red locales, recursos compartidos SMB, SFTP a través de VPN — también detrás de DS-Lite/NAT vía Tailscale.

Canales de comunicación

Telegram, correo electrónico (IMAP/SMTP), mensajes de voz (STT/TTS) — el agente responde donde el empleado pregunta.

Compliance & Seguridad

Construido para auditores,
no contra ellos.

Cada decisión arquitectónica en AIMOS se tomó bajo el aspecto de la conformidad regulatoria — no se adaptó posteriormente.

RGPD / GDPR

Localidad de datos completa en el SovereignNode. Ningún byte abandona su servidor — a menos que usted lo permita explícitamente. PII-Vault anonimiza llamadas API externas automáticamente.

ISO 9001 & 27001

Registro completo de todas las acciones de los agentes. Cada llamada a herramienta, cada solicitud LLM, cada decisión se documenta con marca temporal e ID de sesión.

Seguridad de auditoría

Human-in-the-Loop como principio arquitectónico: Ningún LLM tiene acceso directo a nivel de transacción. Cada operación relevante para la contabilidad requiere aprobación humana.

Conforme a GoBD

Clara separación de borrador y registro definitivo. Los agentes preparan asientos contables — el registro definitivo solo lo realizan operadores autorizados en el sistema destino.

Demostrable mediante Audit-Logs · PII-Vault Traces · Session-IDs · Inspección del código fuente
